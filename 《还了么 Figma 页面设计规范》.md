# 《还了么 Figma 页面设计规范》

Version：V1.0 MVP  
适用端：微信小程序  
设计基准：iPhone 6 / 375px / 750rpx  
依据：《还了么 PRD V1.0》《还了么产品结构图谱》《还了么 UI Design System V1.0》《还了么设计文档》

---

## 1. 文档目标

本规范用于指导设计师在 Figma 中完成《还了么》微信小程序 MVP 高保真页面设计稿。设计重点不是制造金融工具感，而是把“借还约定”设计成一种清晰、温和、可信的关系维护体验。

本次覆盖 7 个 MVP 页面：

| 页面 | 类型 | 默认主题 | 核心目标 |
| --- | --- | --- | --- |
| 首页 | Tab 页面 | 暖橘日常 | 展示用户状态、快捷入口、我的约定 |
| 出手相助 | 二级表单页 | 暖橘日常 | 创建借给好友的约定 |
| 江湖救急 | 二级表单页 | 樱桃粉关怀 | 创建向好友求助的请求 |
| 约定详情 | 二级详情页 | 契约清简 | 查看约定信息并执行状态操作 |
| 延期申请 | 二级表单页 | 契约清简 | 提交新的还款日期与延期原因 |
| 消息中心 | Tab 页面 | 暖橘日常 | 查看系统、约定、成长消息 |
| 个人中心 | Tab 页面 | 暖橘日常 | 展示个人成长、数据与功能入口 |

暂不纳入本次高保真页面：账本、报表、年度报告、API、数据库 ER。

## 3. 三主题风格系统

三种主题均为全局换肤能力。主题切换只改变颜色、插画氛围、图标点缀和轻背景，不改变页面结构、字段顺序、状态逻辑和主要操作位置。

### 3.1 主题定义

| 主题 | Figma Mode | 关键词 | 默认使用 |
| --- | --- | --- | --- |
| 暖橘日常 | `Theme/Orange` | 温暖、亲和、日常信任 | 首页、出手相助、消息中心、个人中心 |
| 樱桃粉关怀 | `Theme/Pink` | 柔和、坦诚、关系修复 | 江湖救急，也可用于求助类空状态 |
| 契约清简 | `Theme/Contract` | 清晰、克制、可信记录 | 约定详情、延期申请、凭证确认 |

## 4. 全局组件规范

### 4.1 导航栏

| 项目 | 规范 |
| --- | --- |
| 高度 | 88rpx，不含状态栏 |
| 标题 | 36rpx，600，居中 |
| 返回按钮热区 | 88rpx x 88rpx |
| 背景 | Tab 页面可同页面背景；二级页面建议白色或主题浅底 |
| 右侧操作 | 仅保留必要操作，如帮助、更多；表单页默认不放复杂操作 |

### 4.2 Tab Bar

| Tab | 图标建议 | 文案 | 选中 |
| --- | --- | --- | --- |
| 首页 | Home / Sprout Home | 首页 | 当前主题 `Primary` |
| 消息 | Message Circle | 消息 | 当前主题 `Primary`，未读显示 Badge |
| 我的 | User Circle | 我的 | 当前主题 `Primary` |

Tab Bar 高度为 `112rpx + safe-area`，图标 48rpx，文字 22rpx，顶部 1rpx 分割线使用 `BorderLight`。

### 4.3 卡片

| 项目 | 规范 |
| --- | --- |
| 背景 | `Theme/*/Surface` |
| 圆角 | 24rpx；列表紧凑卡片可用 20rpx |
| 边框 | 1rpx `Theme/*/BorderLight` |
| 阴影 | `0 8rpx 32rpx rgba(31,41,51,0.04)` |
| 内边距 | 32rpx |
| 间距 | 卡片之间 24rpx 或 32rpx |

### 4.4 按钮

| 类型 | 高度 | 圆角 | 样式 | 使用 |
| --- | ---: | ---: | --- | --- |
| Primary | 96rpx | 24rpx | 当前主题 `ButtonBg`，白字 | 发送约定、发起求助、提交申请 |
| Secondary | 88rpx | 24rpx | 当前主题 `SoftBg`，深色文字 | 取消、再看看、查看说明 |
| Ghost | 80rpx | 20rpx | 透明底，1rpx 边框 | 卡片内轻操作 |
| Danger | 88rpx | 24rpx | `#FFF0F0` 底，`#B72E2E` 字 | 拒绝、删除 |
| Text | 64rpx | 16rpx | 无底色，主题色文字 | 修改、展开、查看更多 |

按钮文案避免金融和催收表达。使用“发送约定”“发送求助”“温馨提醒”“确认完成”“暂不确认”，不使用“放款”“贷款提交”“催款”“结清债务”。

### 4.5 表单

| 项目 | 规范 |
| --- | --- |
| 表单布局 | 单列纵向，分组卡片承载 |
| 标签 | 28rpx，500，`TextPrimary` |
| 必填标记 | 柔和错误色，不使用强红大面积提醒 |
| 输入框 | 高度 88rpx，圆角 20rpx，背景 `SurfaceMuted` |
| 文本域 | 最小高度 176rpx，显示字数统计 |
| 上传组件 | 图片缩略图 144rpx x 144rpx，圆角 16rpx |
| 错误提示 | 字段下方 24rpx Caption，使用 `Error 700` |

### 4.6 弹窗与 Toast

确认弹窗宽度为 `622rpx`，圆角 `32rpx`，内边距 `48rpx`。标题 36rpx，正文 28rpx，按钮横排或纵排根据文案长度决定。

Toast 宽度自适应，最大 `622rpx`，圆角 `24rpx`，背景 `rgba(31,41,51,0.9)`，显示 1500ms 到 2200ms。

## 5. 页面高保真设计稿说明

## 5.1 首页

### 页面定位

首页是用户进入小程序后的默认工作台，承担“看状态、发起约定、处理待办”的任务。默认使用暖橘日常主题，整体要温暖但不促销，信息层级要安静清楚。

### Frame 规格

| 项目 | 规范 |
| --- | --- |
| Frame | `HLM / MVP / 01 首页` |
| 尺寸 | 750rpx x 自适应滚动 |
| 背景 | `Theme/Orange/PageBg` |
| 布局 | 顶部导航 + 滚动内容 + Tab Bar |
| 页面边距 | 32rpx |
| 底部 Padding | Tab Bar 上方至少 160rpx |

### 页面模块

1. 顶部导航
   - 标题：`还了么`
   - 右侧可放 48rpx 消息/设置图标，热区 88rpx。

2. 用户信息卡
   - 卡片宽度 686rpx，圆角 32rpx，内边距 32rpx。
   - 左侧头像 96rpx，昵称 32rpx。
   - 显示信誉分：`信芽 88分`，使用蓝色 `#3B82F6` 小徽标。
   - 显示成长等级：`Lv2 热心市民`，使用橘色/成功色进度条。
   - 右侧签到按钮：高度 64rpx，圆角 32rpx；已签到显示 `已签到`，未签到显示 `签到`。

3. 快捷入口区
   - 两列等宽，每个入口宽 `331rpx`，高度 `184rpx`。
   - 出手相助入口使用暖橘：标题 `出手相助`，说明 `给朋友撑一把伞`。
   - 江湖救急入口使用樱桃粉：标题 `江湖救急`，说明 `有困难坦诚开口`。
   - 图标尺寸 64rpx，建议使用手掌/叶芽、对话气泡/星光。

4. 我的约定
   - 区块标题：`我的温暖约定`
   - 右侧筛选入口：`全部`，点击展开状态筛选。
   - 状态筛选 chips：`全部`、`待确认`、`待还款`、`待完成`、`已逾期`、`已完成`。

5. 约定卡片列表
   - 卡片组件：`Card/Agreement`
   - 折叠态展示：状态、对方昵称脱敏、金额、还款日/倒计时、缘由摘要、主操作。
   - 展开态展示：金额、完整日期、延期记录、借款缘由、提醒语、凭证缩略图、状态操作。
   - 逾期卡片只使用柔和错误浅底与状态标签，不做强红警报。

6. 空状态
   - 组件名：`Empty/HomeNoAgreement`
   - 插画：叶芽、两张轻卡片、暖色对话气泡。
   - 标题：`暂无约定`
   - 描述：`第一份温暖，可以从这里开始`
   - 按钮：`立即出手`、`发起求助`

7. 合规提示
   - 页面底部显示小字：`本记录仅用于约定提醒与情谊守护，不提供借贷、担保或催收服务。`
   - 字号 24rpx，颜色 `TextTertiary`。

### 三主题适配

| 主题 | 首页表现 |
| --- | --- |
| 暖橘日常 | 默认稿；用户卡和入口使用橘色浅底，强调日常亲和 |
| 樱桃粉关怀 | 页面背景转粉色浅底，江湖救急入口更突出，适合求助引导版本 |
| 契约清简 | 背景转灰白，约定卡片更克制，入口减少装饰，适合信息密度更高的版本 |

### 交互状态

| 操作 | 反馈 |
| --- | --- |
| 点击出手相助 | 跳转 `HLM / MVP / 02 出手相助` |
| 点击江湖救急 | 跳转 `HLM / MVP / 03 江湖救急` |
| 点击约定卡片 | 跳转约定详情；若在首页内展开，展开动画 180ms |
| 点击温馨提醒 | Toast：`已发送温馨提醒` |
| 点击签到 | Loading 后变为 `已签到`，展示成长分反馈 |
| 空状态按钮 | 分别跳转创建页 |

### Figma 标注

使用 Auto Layout 纵向排布，页面内容容器命名 `PageContent/Home`。约定卡片建立 variants：`collapsed`、`expanded`、`empty`、`loading`。状态标签使用 `Tag/Status` variants。

## 5.2 出手相助

### 页面定位

出手相助用于创建“我借给好友”的约定。页面语气要像“帮朋友记录清楚”，不是“出借审批”。默认使用暖橘日常主题。

### Frame 规格

| 项目 | 规范 |
| --- | --- |
| Frame | `HLM / MVP / 02 出手相助` |
| 尺寸 | 750rpx x 自适应滚动 |
| 背景 | `Theme/Orange/PageBg` |
| 顶部导航 | 标题 `出手相助`，左侧返回 |
| 底部操作 | 固定底部按钮区，高度 144rpx + safe-area |

### 页面模块

1. 顶部氛围区
   - 高度约 220rpx。
   - 标题：`给朋友撑一把伞`
   - 描述：`把金额、日期和约定说清楚，关系就少一点尴尬。`
   - 使用暖橘浅底和叶芽/手掌线性图标。

2. 表单卡片
   - 字段顺序固定：
     - `借款人`：选择微信好友，占位 `选择需要帮助的朋友`
     - `金额`：金额输入，占位 `请输入金额`
     - `还款日期`：日期选择器，占位 `选择约定归还日期`
     - `借款缘由`：文本域，占位 `简单说明这笔周转的缘由`
     - `辅助凭证`：上传组件，可选，最多 3 张
     - `提醒语`：文本域，占位 `写一句只有对方可见的温暖提醒`
   - 金额输入使用大号数字，单位 `¥` 放在输入框左侧。
   - 凭证提示：`聊天记录、合同或其他说明材料可作为辅助凭证，仅双方可见。`

3. 可信提示卡
   - 背景 `Theme/Orange/SoftBg`。
   - 文案：`还了么只记录约定和提醒，不提供资金、担保或催收服务。`

4. 底部固定操作区
   - 左按钮：`取消`，Secondary。
   - 右按钮：`发送约定`，Primary。
   - 两按钮间距 24rpx，右按钮占更大宽度。

### 三主题适配

| 主题 | 出手相助表现 |
| --- | --- |
| 暖橘日常 | 默认稿；主按钮和顶部氛围使用橘色 |
| 樱桃粉关怀 | 可用于更柔和版本，标题改为 `把帮助说得更温柔`，但字段不变 |
| 契约清简 | 表单边框更明确，适合偏记录版本，减少插画面积 |

### 交互状态

| 状态 | 设计 |
| --- | --- |
| Default | 所有字段为空，主按钮可见但提交时校验 |
| Focus | 输入框边框使用主题 Primary |
| Error | 字段下方显示错误，如 `请填写金额` |
| Uploading | 缩略图位置显示进度，提交按钮禁用 |
| Submit Loading | 主按钮文案 `正在发送` |
| Success | Toast `已发送约定，等待对方确认`，进入 WAIT_CONFIRM |

### Figma 标注

表单组件命名：`Form/FriendPicker`、`Form/AmountInput`、`Form/DatePicker`、`Form/TextArea`、`Form/VoucherUpload`。底部操作命名 `ActionBar/FormSubmit`，设置 fixed prototype。

## 5.3 江湖救急

### 页面定位

江湖救急用于创建“我向好友求助”的请求。页面要降低开口压力，强调坦诚、感谢和按约归还。默认使用樱桃粉关怀主题。

### Frame 规格

| 项目 | 规范 |
| --- | --- |
| Frame | `HLM / MVP / 03 江湖救急` |

## 5.4 约定详情

### 页面定位

约定详情是全流程中最可信的信息页。它需要清楚展示金额、双方、状态、日期、凭证和延期记录，并根据状态给出可执行操作。默认使用契约清简主题。

### Frame 规格

| 项目 | 规范 |
| --- | --- |
| Frame | `HLM / MVP / 04 约定详情` |
| 尺寸 | 750rpx x 自适应滚动 |
| 背景 | `Theme/Contract/PageBg` |
| 顶部导航 | 标题 `约定详情`，左侧返回，右侧更多 |
| 底部操作 | 根据状态展示固定操作按钮 |

### 页面模块

1. 状态摘要卡
   - 顶部放状态标签，如 `待还款`。
   - 金额使用 Amount Large：`¥2,000`
   - 日期信息：`约定归还日 2026-07-05`
   - 倒计时/逾期：`还有 3 天` 或 `已逾期 2 天`

2. 双方信息
   - 左右结构：出借人、借款人。
   - 头像 80rpx，昵称脱敏显示，如 `李*`。
   - 当前用户可用 `我` 标签标识。

3. 约定内容
   - 字段：借款缘由、提醒语。
   - 文本完整展示，长文本可折叠，默认展示 4 行。

4. 凭证预览
   - 标题：`辅助凭证`
   - 缩略图 144rpx，最多横向 3 张，更多显示 `+N`。
   - 提示：`凭证仅约定双方可见。`

5. 延期记录
   - 无记录时显示：`暂无延期记录`
   - 有记录时按时间倒序展示：
     - `申请延期至 2026-07-10`
     - `原因：项目回款延迟`
     - `状态：已同意 / 已拒绝 / 待确认`

6. 时间信息
   - 创建时间、更新时间。
   - 使用 Caption，弱化展示。

7. 底部状态操作
   - 按状态显示不同按钮组。

### 状态操作规则

| 状态 | 主要按钮 | 次要按钮 |
| --- | --- | --- |
| WAIT_CONFIRM | `确认约定` | `暂不确认` |
| WAIT_REPAY | `标记已还款` 或 `温馨提醒` | `申请延期` |
| OVERDUE | `标记已还款` | `申请延期`、`温馨提醒` |
| APPLY_DELAY | 出借人：`同意延期` | `暂不确认` |
| WAIT_FINISH_CONFIRM | 出借人：`确认完成` | `还没收到` |
| FINISHED | `查看凭证` | 无 |
| REJECT | 无主按钮 | `返回首页` |
| EXPIRED | 无主按钮 | `重新创建` |

### 三主题适配

| 主题 | 约定详情表现 |
| --- | --- |
| 暖橘日常 | 可用于首页跳入后的温暖详情，按钮更亲和 |
| 樱桃粉关怀 | 适合求助约定详情，提示文案更柔和 |
| 契约清简 | 默认稿；信息密度更高，状态与凭证更清晰 |

### 交互状态

| 操作 | 反馈 |
| --- | --- |
| 查看凭证 | 打开图片预览，支持左右滑动 |
| 申请延期 | 跳转 `HLM / MVP / 05 延期申请` |
| 确认/拒绝类操作 | 先弹确认弹窗，再提交 |
| 温馨提醒 | Toast `已发送温馨提醒` |
| 确认完成 | Success Toast，状态变为 FINISHED |

### Figma 标注

建立 `Detail/StatusSummary`、`Detail/PeoplePair`、`Detail/VoucherPreview`、`Detail/DelayTimeline`、`ActionBar/StatusActions`。状态操作按钮用 variants 管理，不为每个状态复制散乱按钮。

## 5.5 延期申请

### 页面定位

延期申请用于借款人提出新的归还日期和原因。页面要表达“说明情况、等待确认”，不制造逃避或压力感。默认使用契约清简主题。

### Frame 规格

| 项目 | 规范 |
| --- | --- |
| Frame | `HLM / MVP / 05 延期申请` |
| 尺寸 | 750rpx x 自适应滚动 |
| 背景 | `Theme/Contract/PageBg` |
| 顶部导航 | 标题 `延期申请`，左侧返回 |
| 底部操作 | 固定主按钮 `提交申请` |

### 页面模块

1. 当前约定摘要
   - 展示金额、原还款日期、当前状态。
   - 用浅灰卡片承载，避免用户填写时忘记上下文。

2. 表单卡片
   - `新还款日期`：日期选择器，占位 `选择新的归还日期`
   - `延期原因`：文本域，占位 `说明需要延期的原因`
   - 字数建议 10 到 120 字，右下显示字数。

3. 说明提示
   - 文案：`提交后将等待对方确认。对方同意后，新的还款日期才会生效。`

4. 底部按钮
   - 主按钮：`提交申请`
   - 禁用态：新日期或原因为空时可提交校验，也可置灰，推荐置灰并保留提示。

### 三主题适配

| 主题 | 延期申请表现 |
| --- | --- |
| 暖橘日常 | 更温和，适合普通周转说明 |
| 樱桃粉关怀 | 更强调表达难处，适合求助场景 |
| 契约清简 | 默认稿；突出日期、原因和确认机制 |

### 交互状态

| 操作 | 反馈 |
| --- | --- |
| 点击提交 | 弹窗确认：`提交后将等待对方确认，确定提交吗？` |
| 确认提交 | Loading：`正在提交` |
| 成功 | Toast `延期申请已提交`，返回约定详情，状态 APPLY_DELAY |
| 失败 | Toast `提交失败，请重试` |

### Figma 标注

组件命名 `Form/DelayDatePicker`、`Form/DelayReason`、`Modal/DelayConfirm`。底部主按钮为 fixed，内容区底部 padding 不小于 176rpx。

### 页面定位

消息中心承接系统通知、约定提醒和成长通知。默认使用暖橘日常主题，消息应可快速扫描，重点提醒可直达约定详情。

   - 展示 `12 条未读`
   - 可加小提示：`新的约定提醒会出现在这里`

2. 分类 Tabs
   - `全部`、`系统通知`、`约定提醒`、`成长通知`

   - 组件：`List/MessageItem`
   - 每条消息包含：类型图标、标题、摘要、时间、未读点。

   - 列表顶部或右上角提供 `全部已读`。

| 全部已读 | 未读点消失，Toast `已全部标为已读` |

## 7. 设计验收清单

### 7.1 内容一致性

| 检查项 | 标准 |
| --- | --- |
| 页面范围 | 只覆盖 7 个 MVP 页面 |
| 表单字段 | 与 PRD 字段一致，不增删核心必填项 |
| 状态体系 | 使用 WAIT_CONFIRM、WAIT_REPAY、OVERDUE、APPLY_DELAY、WAIT_FINISH_CONFIRM、FINISHED、REJECT、EXPIRED |
| 产品边界 | 明确不提供借贷、资金、担保、催收服务 |
| 文案语气 | 温暖、可信，避免金融审批和催收表达 |

### 7.2 小程序适配

| 检查项 | 标准 |
| --- | --- |
| 单位 | 核心尺寸均使用 rpx |
| 画板 | 750rpx 宽，等效 375px |
| 点击热区 | 不小于 88rpx |
| 安全区 | 顶部导航、底部 Tab、吸底按钮均预留 safe-area |
| 滚动内容 | 吸底按钮页面底部 padding 不小于 176rpx |

### 7.3 三主题验收

| 检查项 | 标准 |
| --- | --- |
| 全局换肤 | 七个页面均可切换 Orange、Pink、Contract |
| 结构稳定 | 主题切换不改变信息架构 |
| 暖橘日常 | 不做金融金色、不做促销风 |
| 樱桃粉关怀 | 不娱乐化，不弱化关键信息 |
| 契约清简 | 不压迫、不催收化，保持可信清楚 |

### 7.4 Figma 落稿验收

| 检查项 | 标准 |
| --- | --- |
| Frame 命名 | 使用 `HLM / MVP / 01 首页` 格式 |
| Variables | 颜色通过主题变量管理 |
| Components | 卡片、按钮、表单、标签、消息项均组件化 |
| Variants | 状态标签、按钮状态、卡片展开/折叠、消息已读/未读均有 variants |
| Prototype | 核心跳转关系完整 |
